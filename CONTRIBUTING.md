# Contributing to Stacks Docs

Thank you for your interest in contributing to the Stacks documentation! We welcome contributions from the community to help make our documentation better for everyone.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](./CODE_OF_CONDUCT.md).

## How to Contribute

### Reporting Issues

If you find a typo, a missing guide, or incorrect information, please [open an issue](https://github.com/Earnwithalee7890/stacks-docs/issues/new) on GitHub.

### Submitting Pull Requests

1. **Fork the Repository**: Click the "Fork" button on the top right of the repository page.
2. **Clone your Fork**: `git clone https://github.com/YOUR_USERNAME/stacks-docs.git`
3. **Create a Branch**: `git checkout -b my-new-feature`
4. **Make Changes**: Edit the markdown files in the `docs/` directory.
5. **Commit Changes**: `git commit -m "Add new guide for..."`
6. **Push to GitHub**: `git push origin my-new-feature`
7. **Submit a Pull Request**: Go to the original repository and click "Compare & pull request".

## Style Guide

- **Format**: We use Markdown for all documentation.
- **Headers**: Use sentence case for headers (e.g., "How to run a node").
- **Links**: Use relative links for internal documentation.
- **Images**: Place images in the `.gitbook/assets/` folder (or equivalent) if applicable.

## Questions?

If you have any questions, feel free to ask in the issues or contact the maintainers.
